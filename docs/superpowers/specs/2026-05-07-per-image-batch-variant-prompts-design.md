# QiuAi 图片级差异化专属提示词设计

日期：2026-05-07

## 背景

QiuAi 当前的 `套图设计` 与 `套图生成` 都支持多组输出，但多组任务默认会重复使用同一套提示词。

现状：

- `套图设计`：每轮复用同一套 `globalPrompt + imageAssignment.prompt`
- `套图生成`：每轮复用同一套 `globalPrompt + promptAssignment.prompt`
- `batchCount` 只控制输出组数，不控制每组提示词差异

直接结果是：

- 多批次结果相似度高
- 用户虽然设置了多组，但没有真正拉开各组创意差异
- 相似度问题不是模型随机性不足，而是输入提示词本身在多组之间没有变化

本次设计目标是用最小复杂度解决这个问题，不引入新的标签体系，不引入新的提示词池，而是在每张图片自己的卡片里增加一个开关，允许该图片按组使用不同的专属提示词。

## 目标

### 业务目标

- 让多组输出时，每一组可以真正使用不同提示词
- 差异化配置粒度落到“每张图片”而不是整个任务
- 不开启差异化时，完全保持现有逻辑不变

### 产品目标

- 让用户一眼能看懂差异化功能
- 不新增新的标签库学习成本
- 差异化交互尽量复用用户已经熟悉的“专属提示词”概念

### 非目标

- 本次不做差异化标签
- 本次不做组级提示词模板
- 本次不做自动随机差异化
- 本次不做一键批量复制差异化配置

## 方案选择

## 方案 A：差异化标签 / 差异化提示词池

形式：

- 为每张图片增加按组标签选择
- 或者增加一套差异化提示词池机制

优点：

- 理论上更灵活

缺点：

- 用户理解成本高
- 配置复杂
- 会把现有简单的“专属提示词”路径变成多层概念

结论：不采用。

## 方案 B：差异化开关 + 多个专属提示词

形式：

- 每张图片增加一个 `差异化` 勾选框
- 勾选后，按 `batchCount` 自动展开：
  - `专属提示词1`
  - `专属提示词2`
  - `专属提示词3`
- 不勾选则保持当前单个 `专属提示词`

优点：

- 用户最容易理解
- 和现有“专属提示词”概念完全连续
- 后端拼接逻辑清楚
- 不会引入新的数据体系

缺点：

- 当批次很大时，输入框数量会增多

结论：采用。

## 最终设计

## 一、适用范围

本次改动只作用于：

- `套图设计`
- `套图生成`

不影响：

- `单图测试`
- `单图设计`
- `提示词库`

## 二、交互设计

## 套图设计

每个 `imageAssignment` 图片卡片里，当前已有：

- 图片勾选框
- 图片类型
- 单张比例
- 单张模型
- 专属提示词

本次新增：

- 一个 `差异化` 勾选框

位置要求：

- 跟在当前图片勾选框后面
- 保持这是当前图片自己的设置，不放到任务顶部

行为规则：

- 未勾选 `差异化`
  - 继续显示原有单个 `专属提示词`
  - 多组任务复用这一个提示词
- 勾选 `差异化`
  - 隐藏原有单个 `专属提示词`
  - 根据 `batchCount` 动态显示多个输入框
  - 例如 `batchCount = 3` 时显示：
    - `专属提示词1`
    - `专属提示词2`
    - `专属提示词3`

## 套图生成

每个 `promptAssignment` 卡片里，当前已有：

- 图片类型
- 当前提示词

本次新增：

- 一个 `差异化` 勾选框

位置要求：

- 跟当前条目自身的选择控件放在一起
- 语义上表示“这一张输出图是否按组使用不同提示词”

行为规则与 `套图设计` 一致：

- 不勾选：继续使用单个原始提示词
- 勾选：根据 `batchCount` 展开多个 `专属提示词1/2/3...`

## 三、数据结构设计

本次不引入复杂对象结构，只增加两个简单字段。

## 套图设计

在每个 `imageAssignment` 下新增：

```js
{
  prompt: '',
  differentialEnabled: false,
  batchPrompts: []
}
```

字段含义：

- `prompt`
  - 原有单个专属提示词
- `differentialEnabled`
  - 是否开启差异化
- `batchPrompts`
  - 开启差异化时按组使用的提示词数组

## 套图生成

在每个 `promptAssignment` 下新增：

```js
{
  prompt: '',
  differentialEnabled: false,
  batchPrompts: []
}
```

字段含义与 `套图设计` 一致。

## 四、状态同步规则

## 未开启差异化

- `differentialEnabled = false`
- `batchPrompts` 可以为空数组
- 实际提交仍读取原有 `prompt`

## 开启差异化

- `differentialEnabled = true`
- 提示词来源改为 `batchPrompts`
- 原有 `prompt` 暂不生效，但保留在草稿中，不自动清空

这样做的好处是：

- 用户取消勾选 `差异化` 后，仍可回到原来单个提示词
- 用户重新勾选 `差异化` 后，之前填写过的 `batchPrompts` 也可以继续保留

## batchCount 增加

当 `batchCount` 从 2 改为 4 时：

- 若某张图片开启了差异化，则它的 `batchPrompts` 自动补齐到 4 项
- 新增项默认为空字符串

## batchCount 减少

当 `batchCount` 从 4 改为 2 时：

- UI 只显示前 2 个 `专属提示词`
- 数据层可直接截断为前 2 项

本次推荐直接截断，保持实现简单。

## 五、提示词拼接规则

## 套图设计

### 未开启差异化

每张图每组都使用：

```text
globalPrompt
+ assignment.prompt
```

### 开启差异化

第 N 组使用：

```text
globalPrompt
+ assignment.batchPrompts[N - 1]
```

即：

- 第 1 组读 `batchPrompts[0]`
- 第 2 组读 `batchPrompts[1]`
- 第 3 组读 `batchPrompts[2]`

## 套图生成

### 未开启差异化

每张输出图每组都使用：

```text
globalPrompt
+ promptAssignment.prompt
```

### 开启差异化

第 N 组使用：

```text
globalPrompt
+ promptAssignment.batchPrompts[N - 1]
```

## 反向提示词

反向提示词逻辑不变，仍然在最终正向提示词拼好之后追加。

## 六、界面显示规则

为避免用户混淆，差异化开启后不应同时展示两套提示词入口。

规则如下：

- `differentialEnabled = false`
  - 显示单个 `专属提示词`
  - 隐藏 `专属提示词1/2/3`
- `differentialEnabled = true`
  - 隐藏单个 `专属提示词`
  - 显示多个 `专属提示词1/2/3`

这样能明确表达：

- 只有一套输入来源在生效

## 七、兼容规则

为了兼容旧草稿与旧数据：

- 老数据没有 `differentialEnabled`
  - 归一化时补为 `false`
- 老数据没有 `batchPrompts`
  - 归一化时补为 `[]`

因此：

- 老版本草稿不会报错
- 老用户不需要额外迁移操作

## 八、结果回显

结果展示区下方已有“发送提示词”文本框。

本次需要保证这里显示的是每张结果图当时真正发送的完整提示词。

这意味着：

- 未开启差异化时，显示 `globalPrompt + prompt`
- 开启差异化时，显示 `globalPrompt + 当前组对应的 batchPrompts[n]`

这样用户能直接验证：

- 第 1 组和第 2 组的提示词是否真的不同
- 差异化是否生效

## 九、测试重点

应补充以下测试：

1. 老草稿没有 `differentialEnabled` 和 `batchPrompts` 时可正常归一化
2. `batchCount` 增加时，开启差异化的图片会自动补齐 `batchPrompts`
3. `batchCount` 减少时，`batchPrompts` 会按新批次截断
4. `套图设计` 开启差异化后，不同组会使用不同 `batchPrompts`
5. `套图设计` 未开启差异化时，仍走原有 `prompt`
6. `套图生成` 开启差异化后，不同组会使用不同 `batchPrompts`
7. `套图生成` 未开启差异化时，仍走原有 `prompt`
8. 结果展示里的 `promptFinal` 能正确反映不同组提示词

## 十、范围控制

本次实现只包含：

- 差异化勾选框
- 多个 `专属提示词1/2/3` 输入框
- 前后端草稿归一化
- 最终提示词拼接
- 结果回显和测试

本次明确不包含：

- 标签差异化
- 自动随机差异化
- 差异化模板库
- 批量复制到其他图片

## 结论

本次采用“图片级差异化开关 + 多个专属提示词”的方案。

核心原则是：

- 差异化只在用户主动勾选时生效
- 不勾选时完全保持原逻辑
- 勾选后按批次展开多个提示词输入框
- 每一组真正读取不同的专属提示词

这是当前最容易理解、最容易实现、也最不容易把用户用法复杂化的方案。
