1. 软件整体界面说明

QiuAi 的主界面可以理解为 4 个区域：

顶部工具栏

用于查看授权状态、执行一键清理、切换主题，以及查看服务方联系方式。

左侧侧边栏

主导航区，包含：工作台、单图测试、单图设计、套图设计、套图生成、模型价格、提示词库。

中间主工作区

用于填写参数、上传图片、选择模型、设置提示词，以及查看生图结果。

右侧辅助区

在工作台中显示任务队列，在生图相关页面中显示结果导出。

2. 使用前先了解

2.1 授权与配置

软件正常打开并显示“已激活”后即可使用。API 已由服务方预配置，普通用户不需要自行修改。

2.2 一键清理

顶部有“一键清理”按钮。该功能会重置参数草稿、效果展示和日志缓存，但不会删除已导出的结果文件、提示词库、已保存目录和 API 配置。

如果当前还有进行中的任务，一键清理可能会失败。

2.3 任务进度

提交任务后，中间“效果展示”区域会显示任务状态、当前进度、当前组、组内进度以及失败原因。任务异常时，请优先看这里。

3. 工作台

“工作台”适合用来总览软件当前运行状态。

可以看到什么

  • 单图测试统计
  • 单图设计统计
  • 套图设计统计
  • 套图生成统计
  • 网络监控
  • 积分仪表盘
  • 积分消息记录
  • 服务配置状态
  • 用户主机信息
  • 任务队列

适合什么时候用

  • 想快速查看最近任务情况
  • 想确认积分是否充足
  • 想查看网络请求是否正常
  • 想结束卡住的任务
右侧“任务队列”会显示任务编号、生成时间、当前进度和当前错误。对于“等待中”“进行中”“待确认”的任务，可以点击“结束任务”主动停止。

4. 单图测试

“单图测试”适合做模型对比。它会基于同一张图片、同一套提示词，输出多个模型的效果，方便你判断哪个模型更适合当前场景。

操作步骤

  1. 进入左侧“单图测试”
  2. 上传 1 张测试图片
  3. 在“提示词输入区域”填写统一提示词
  4. 如有需要，在“补充说明”里写补充要求
  5. 选择输出比例
  6. 确认对比模型
  7. 点击“提交任务”

5. 单图设计

“单图设计”适合处理单张图片的正式生成任务。如果你已经明确要用哪个模型，通常直接在这里做。

操作步骤

  1. 进入左侧“单图设计”
  2. 上传参考图片
  3. 在“提示词输入区域”输入本次设计要求
  4. 在“补充说明”中补充风格、材质、光线、构图等细节
  5. 选择设计模型
  6. 选择输出比例
  7. 点击“提交任务”

6. 套图设计

“套图设计”适合多张图片一起生成，每张图可以单独设置，是当前批量工作的核心板块之一。

操作步骤

  1. 上传一整套图片素材
  2. 填写“全局主提示词”
  3. 选择模型
  4. 选择“反向提示词”模板
  5. 设置“生成组数”
  6. 对每一张图片分别设置是否参与生成、是否差异化、图片类型、单张比例、单张模型和专属提示词
  7. 点击“提交任务”

差异化怎么理解

如果某张图片勾选了“差异化”，系统会根据你设置的生成组数，自动展开“专属提示词1、专属提示词2、专属提示词3……”这样的多条输入框。这样每一组生成时，这张图都能使用不同的专属提示词，减少结果过于相似的问题。

7. 套图生成

“套图生成”适合基于 1 张参考图，批量生成多张不同画面，更适合同一主体下的多图扩展。

操作步骤

  1. 上传 1 张参考图
  2. 输入“全局风格提示词”
  3. 选择模型
  4. 选择“反向提示词”模板
  5. 设置“生成数量”
  6. 设置“批次”
  7. 逐张填写提示词配置
  8. 点击“提交任务”

8. 模型价格

“模型价格”页主要用于查看积分充值档位和各模型的积分消耗。建议在大批量任务前先看这一页，避免积分不足导致任务中断。

9. 提示词库

“提示词库”用于管理正向提示词模板、负向提示词模板，以及查看提示词风险提醒。

正向提示词

可新建、编辑、保存正向模板，也可以删除自定义模板。系统模板和自定义模板统一显示在列表中。

负向提示词

当前默认内置“电商通用”“电商模特”“电商静物”3 套模板，用于限制水印、变形、模糊、杂乱背景等问题。

提示词格式

按不同场景给出推荐字符数和写法重点，适合新手快速上手。

风险提示区

右侧展示违禁提示词和警告提示词，帮助你避开高风险表达。

10. 结果导出

在“单图测试”“单图设计”“套图设计”“套图生成”页面的右侧，会显示“结果导出”区域。

  • 勾选要导出的结果文件夹
  • 打开输出目录
  • 删除结果文件夹
  • 批量下载

顶部有“自动删除”勾选项，默认开启。批量下载成功后，软件会自动删除已下载完成的源结果文件夹，避免文件长期堆积。

11. 常见使用建议

  • 尽量避免使用“和原图一致”“保持原样”“不改动布局”“轻微修改”等高风险表达。
  • 批量任务建议先用 1 组或少量图片测试，确认方向后再放大批次。
  • 如果多组结果相似度高，建议开启差异化，并为不同组填写不同专属提示词。
  • 大批量任务前先看积分，尤其是套图设计、套图生成和多差异化任务。

12. 常见问题

为什么提交后失败了

常见原因包括：服务配置无效、提示词风险过高、积分不足、网络请求异常、批量任务过大。失败时请优先查看顶部消息提示和结果区中的失败原因。

为什么一键清理失败

通常是因为当前还有进行中的任务。建议先到“工作台”结束任务，再执行一键清理。

为什么结果导出里的文件越来越多

如果没有开启“自动删除”，下载后的源结果文件夹会一直保留。建议保持开启。

为什么图片看起来相似

通常是因为多批次使用了重复提示词。建议开启差异化，并为不同组填写不同专属提示词。

13. 推荐使用流程

  1. 先看“模型价格”，确认积分和模型成本
  2. 打开“提示词库”，了解正向模板、负向模板和风险词
  3. 先用“单图测试”做小范围模型对比
  4. 再用“单图设计”或“套图设计”进入正式生成
  5. 生成完成后，在右侧“结果导出”中批量下载
  6. 如任务异常，到“工作台”查看任务队列和错误提示

14. 结语

QiuAi 当前更适合围绕“参考图 + 提示词 + 批量生成”这一工作流使用。建议先把提示词方向、模型、负向模板和小批次测试跑顺，再放大到正式批量任务，这样整体效率和稳定性会更高。
